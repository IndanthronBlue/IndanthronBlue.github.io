SQL｜高级 SQL

高级 SQL

函数和过程

  • SQL提供了类似高级语言的程序结构

    • 如分支结构（if语句）、循环结构（while，for语句等）可以完成复杂的“业务逻辑”，并存储在数据库中，然后在SQL中调用

  • 存储过程(Stored Procedures)

    • 可以将过程存储在数据库中
    • 然后使用调用 (exec) 语句执行它们
    • 允许外部应用程序在不知道内部细节的情况下对数据库进行操作
    • 当规则发生变化时，可以不需要修改其它代码。当参数变化时，不需要重复写代码。

  • 函数和过程

    • 函数和过程允许"业务逻辑"存储在数据库中并从 SQL 语句执行。
    • 这些可以通过SQL的过程组件或外部编程语言（如Java，C或C++）来定义。

声明和调用 SQL 函数和过程

  • 函数的定义：
    • 函数要有函数名、自变量（参数）、返回值和函数体（函数实现的功能）
    • 示例
  • 函数定义之后就可以调用，以下示例给出了在SQL中的调用例子：where子句中调用了所定义的函数
  • SQL也支持返回关系作为结果的函数：table function
    • 通常以表为值的函数可以被看作 带参数的视图(parameterized view)
    • 它通过允许参数把视图的概念更加一般化
    • 注意，使用函数的参数时要加上函数名作为前缀
  • 使用表函数：
  • SQL支持过程，同样先定义后调用。
  • 定义过程需要有过程名、输入参数和输出参数，分别用 in 和 out 关键字标识
  • 过程定义之后，可以在SQL中或嵌入式SQL中用call这个命令调用过程执行
  • SQL允许多个过程同名，只要参数个数不同
    • 名称和参数个数用于唯一标识一个过程
    • SQL也允许多个函数同名，只要函数个数不同就可以

支持过程和函数的语言构造

  • SQL中的结构允许类似高级语言的程序结构，如循环、赋值、分支判断、异常捕获等

    • SQL 标准中处理这些构造的部分称为 持久存储模块(Persistent Storage Module, PSM)

  • 变量通过 declare 语句进行声明，可以是任何合法 SQL 类型。使用 set 语句进行赋值

  • 一个复合语句有 begin ... end 的样式

    • 在 begin 和 end 之间会包含复杂的 SQL 语句
    • 可以在复合语句中声明局部变量

  • SQL: 1999 支持 while 语句和 repeat 语句

  • SQL支持迭代，允许查询的结果重复执行
    • 程序每次获取查询结果的一行，并存入 for 循环变量
    • 语句中 leave 可用来退出循环，而 iterate 表示跳过剩余语句从循环的开始进入下一个元组
  • SQL 程序语言还支持发信号通知 异常条件(exception condition) ，以及声明 句柄(handler) 来处理异常
  • 在 begin 和 end 之间的语句可以执行 signal out_of_classroom_seats 来引发一个异常
  • 这个句柄说明，如果条件发生，将会采取行动终止 begin end 中的语句
  • 另一个可选的动作是 continue
    • 它继续从引发异常的语句的下一条语句开始执行

触发器 Trigger

  • 触发器(Trigger) 是一条语句，当对数据库做修改时，它自动被系统执行
    • 也称为 事件-条件-操作规则(event-condition-action rules, ECA)
    • 触发器允许程序员指定何时进行检查以及在哪些条件下应执行一组操作
    • 触发器允许应用程序监视数据库事件（活动），并在发生指定事件时执行操作
    • 应用trigger时要做两项工作：
      • 说明什么条件下执行触发器：引起触发器的事件（event）和触发器执行必须满足的条件（condition）
      • 触发器执行时触发的动作

SQL 中的触发器

  • 这个例子介绍了 trigger 的定义：Trigger 定义包括 trigger 的名字、触发的事件和条件以及执行的动作

    • 这个 trigger 命名为 timeslot_check1
    • 事件是对数据库表 section 进行 insert 操作
    • trigger执行的条件是当插入 section 表的的新数据的 time_slot_id 不在 time_slot 表中时，由 when 语句指定
    • Trigger执行的操作是 rollback
    • 实现的功能是：当对 section 表进行 insert 操作时，如果增加的新的纪录的 time_slot_id 不在 time_slot 表中，那么这个这个 insert 操作无法成功执行，插入的数据要删掉，所有对数据库的操作 rollback（回滚），恢复插入之前的数值
    • 触发器代码中 for each row 语句可以显式地在每个被插入的行上进行迭代

  • 注意：

    • referencing new row as nrow 这条语句，建立一个临时变量存储新增的数据
    • 还有另一种方式 referencing old row as orow ，建立临时变量存储删掉或修改行的旧数据
    • as 子句后是临时变量名，实际是一个临时表。

  • 定义 trigger 时的 event 和 action：

    • event 是对数据库的修改操作：insert、delete 和 update

      • 可以引用更新前后的属性值
        • referencing old row as: for deletes and updates
        • referencing new row as: for inserts and updates

    • action 根据数据管理需要完成语句

  • trigger 可以在事件发生前也可以在事件发生后触发，可以作为避免非法更新、插入或删除的额外约束

  • trigger 有很多合适的应用场景，例如早期维护物化视图，但是新版本支持自动维护；早期的复制和维护数据，现在则有专门的内置数据库复制工具；

写 trigger 时要特别小心，因为运行期间一个 trigger 错误会导致引发该 trigger 的 action 语句失败，而且一个 trigger 的动作可以引发另一个触发器，最坏情况下，会导致一个无限的触发链
